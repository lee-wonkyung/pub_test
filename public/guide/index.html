01

Figma에서 .fig 파일 추출

디자인 원본을 Claude Design에 올릴 형태로 내보내기
STEP 1 .fig 파일로 내보내기
  1. Figma에서 변환할 화면이 포함된 파일을 연다.
  2. 왼쪽 사이드바 파일 목록(또는 좌상단 메뉴)에서 대상 파일을 우클릭한다.
  3. Save local copy… (로컬 사본 저장)을 선택하면 .fig 파일이 다운로드된다.
  4. 특정 화면만 필요하면, 해당 프레임을 새 파일로 복제한 뒤 그 파일을 .fig로 내보낸다.
왜 복사본인가

원본을 그대로 올리면 미사용 페이지·숨김 레이어·여러 작업 버전까지 함께 해석돼 정확도와 속도가 떨어진다. 또 추출 전 정리(불필요한 레이어 삭제·프레임 분리)를 원본에서 하면 디자이너의 실제 작업 파일이 훼손되고, 팀이 공유·동시 편집 중인 파일과 충돌할 수 있다. 복사본에서 필요한 화면만 남겨 정리하면 원본을 건드리지 않으면서 가볍고 깔끔한 .fig를 만들 수 있다.

대안 — 파일 전체가 아닌 일부만
  • 필요한 프레임만 선택 → 복사 → 새 Figma 파일에 붙여넣기 → 그 파일을 Save local copy…로 내보내면 가벼운 .fig를 만들 수 있다.
  • 불필요한 페이지·숨김 레이어·미사용 컴포넌트를 정리하면 Claude Design의 해석 정확도가 올라간다.

그림 1. Figma에서 .fig 파일 내보내기.

팁

화면 단위로 파일을 나눠 내보내면 한 번에 생성되는 결과물 범위가 명확해져 프롬프트도 단순해지고 결과 편차도 줄어든다. "한 파일 = 한 작업 단위"를 권장.

02

Claude Design에 업로드

추출한 .fig 파일을 입력으로 제공
STEP 2 디자인 파일 올리기
  1. Claude에 접속해 Claude Design(디자인 입력) 기능을 연다.
  2. 새 대화를 시작하고, 첨부(파일 업로드) 버튼으로 STEP 1에서 받은 .fig 파일을 올린다.
  3. 파일이 정상 인식되었는지(미리보기·파일명) 확인한다.
  4. 한 번에 여러 화면을 다룰 경우, 화면별로 파일을 나눠 올리거나 프롬프트에서 화면을 명확히 구분한다.

그림 2. Claude Design에 .fig 파일 업로드.

주의

용량이 큰 파일은 업로드·해석 시간이 길어진다. 미사용 에셋·페이지를 정리한 가벼운 .fig를 올리는 것이 안정적이다.

03

프롬프트 작성 · 마크업 생성

원하는 결과를 명확히 지시해 코드 품질을 끌어올리기

같은 디자인이라도 프롬프트에 따라 결과물 품질이 크게 달라진다. 출력 형식 · 기술 스택 · 제약 조건을 처음부터 명시하면 후속 수정량이 줄어든다.

범용 프롬프트 템플릿

복사해서 대괄호 부분만 바꿔 사용
첨부한 Figma 디자인(.fig)을 코드로 변환해줘. # 출력 형식 - [ HTML + CSS / React + CSS Modules / Vue 등 ] 으로 작성 - CSS는 [ 인라인 / 별도 파일 / CSS 변수 토큰 ] 방식으로 - 반응형: [ 데스크톱 고정 / 모바일 우선 / 둘 다 대응 ] # 디자인 충실도 - 색상·여백·폰트 크기는 디자인 수치를 최대한 그대로 반영 - 레이아웃은 [ 절대좌표 1:1 / Flex·Grid 기반 재구성 ] 으로 - 폰트는 [ Pretendard / 시스템 폰트 / 디자인 지정 ] 사용 # 제약 · 규칙 - 반복되는 요소(버튼·입력폼·카드)는 재사용 컴포넌트로 분리 - 컴포넌트·클래스 네이밍은 [ 케밥케이스 / 카멜케이스 ] 통일 - 이미지·아이콘은 [ 플레이스홀더 SVG / 자리표시 박스 ] 로 처리 - 접근성(label, alt, 시맨틱 태그)을 지킬 것 # 산출물 - 실행 가능한 파일 전체를 zip으로 받을 수 있게 구성

작성 팁

항목 이렇게 (권장) 이러면 곤란
기술 스택 "React + CSS Modules로" 처럼 명시 스택 미지정 → 매번 다른 형태
레이아웃 방식 Flex·Grid 재구성 / 좌표 1:1 중 택1 미지정 → 절대좌표 남발
컴포넌트화 "반복 요소는 컴포넌트로 분리" 지시 중복 마크업 양산
네이밍 규칙을 먼저 못 박기 Button·PrimaryButton 혼재
범위 화면 단위로 끊어 요청 전체 한 번에 → 품질 저하
반복 개선

한 번에 완벽을 노리기보다, 생성된 결과를 보고 "헤더 여백을 디자인대로 24px로", "버튼을 공통 컴포넌트로 묶어줘" 처럼 후속 지시로 다듬는 방식이 효율적이다.

04

zip 파일 다운로드

생성 결과를 로컬 작업 환경으로 가져오기
STEP 4 결과물 내려받기
  1. 생성된 결과 화면에서 다운로드(또는 Export / Download code) 버튼을 찾는다.
  2. zip 파일로 내려받는다. 파일 전체가 폴더 구조 그대로 묶여 있다.
  3. 다운로드한 zip을 작업 폴더에 압축 해제한다.
  4. 구조를 한 번 훑어 본다 — HTML/CSS/JS, 컴포넌트 폴더, 에셋 위치 등.

그림 3. 프롬프트 작성 후 생성된 마크업 미리보기.

예시 — 압축 해제 후 구조
project/ ├─ index.html ├─ styles/ │ ├─ tokens.css # 색상·타이포·간격 토큰 │ └─ global.css ├─ components/ # 재사용 컴포넌트 │ ├─ Button/ │ └─ TextField/ └─ assets/ # 아이콘·이미지(플레이스홀더 가능)
확인

다운로드 직후 결과물은 아이콘·이미지가 플레이스홀더이거나 레이아웃이 디자인과 미세하게 다를 수 있다. 다음 단계(Claude Code)에서 정리한다.

05

Claude Code로 수정 · 정리

프로젝트 컨텍스트에 맞춰 실서비스 코드로 다듬기

Claude Design 결과물은 빠르지만 프로젝트 컨텍스트가 없다(기존 컴포넌트·디자인 시스템 미반영). 압축 해제한 결과물을 기존 프로젝트에 넣고 Claude Code로 통합·정리한다.

STEP 5 코드 통합 · 보정
  1. 압축 해제한 결과물을 작업 프로젝트 폴더에 배치한다.
  2. 해당 폴더에서 터미널을 열고 claude 를 실행한다.
  3. 아래 프롬프트로 기존 컴포넌트 재사용·네이밍 통일·접근성/SEO 보정을 요청한다.
  4. 변경분을 검토하고, 브라우저에서 디자인과 1:1 대조해 마무리한다.

Claude Code 정리 프롬프트 (범용)

방금 추가한 [ 폴더/파일 경로 ]는 Claude Design으로 생성한 마크업이야. 이걸 우리 프로젝트에 맞게 정리해줘. - 기존 [ components/ ]의 Button·TextField 등 공통 컴포넌트가 있으면 새로 만들지 말고 그걸 재사용하도록 교체 - 디자인 토큰( [ tokens.css / theme ] )이 있으면 하드코딩된 색·간격을 토큰으로 치환 - 클래스·컴포넌트 네이밍을 프로젝트 컨벤션( [ 케밥/카멜 ] )에 맞게 통일 - 시맨틱 태그·label·alt 등 접근성 보강, 명백한 SEO 이슈 정리 - 중복 마크업·미사용 스타일 제거 먼저 변경 계획을 알려주고, 동의하면 수정해줘.
Claude Code로 보정할 항목 왜 필요한가
기존 컴포넌트 재사용 중복 컴포넌트(Button / PrimaryButton 혼재) 방지
디자인 토큰 치환 하드코딩된 색·간격을 단일 소스로 정리
네이밍 통일 코드베이스 컨벤션 일관성 확보
접근성 · SEO AI 생성 코드의 누락 보강(검토 필수)
레이아웃 미세 보정 여백·정렬을 디자인 수치와 일치
권장 순서

① 구조·컴포넌트 통합 → ② 토큰·네이밍 정리 → ③ 접근성/SEO → ④ 픽셀 보정 순으로 큰 것부터 다듬으면 재작업이 줄어든다. 각 단계마다 변경 계획을 먼저 확인하는 습관을 권장.

06

완료 체크리스트

배포 전 마지막 점검
  • 디자인과 화면을 1:1로 대조했는가 (색·여백·폰트·정렬)
  • 반복 요소가 공통 컴포넌트로 정리되었는가
  • 하드코딩 값이 디자인 토큰으로 치환되었는가
  • 네이밍이 프로젝트 컨벤션과 일치하는가
  • 접근성(label·alt·시맨틱)·SEO를 확인했는가
  • 아이콘·이미지 플레이스홀더를 실제 에셋으로 교체했는가
  • 불필요한 중복 마크업·미사용 스타일을 제거했는가
기억할 것

AI 생성 코드는 보조 도구다. 빠른 초안 생성에는 탁월하지만, 실서비스 반영 전에는 사람의 검토가 필수다.

APPENDIX A

자주 묻는 문제

.fig가 안 보여요
파일 목록에서 파일을 우클릭 → Save local copy…. 프레임 단위 선택이 아니라 파일 단위로 내보내야 .fig가 생성된다.
업로드가 느려요
미사용 페이지·숨김 레이어·미사용 컴포넌트를 정리한 가벼운 .fig로 다시 시도.
결과가 매번 달라요
기술 스택·레이아웃 방식·네이밍 규칙을 프롬프트에 명시. 화면 단위로 끊어 요청.
컴포넌트가 중복돼요
Claude Code 단계에서 "기존 공통 컴포넌트를 재사용하라"고 명시해 통합.
아이콘이 플레이스홀더예요
정상 동작. 실제 브랜드 에셋으로 교체하거나, MCP 워크플로우에서는 이미지 내보내기로 대체.
APPENDIX B

용어

.fig
Figma 파일의 로컬 사본 포맷. Save local copy…로 내보낸다.
Claude Design
디자인을 입력해 UI 코드를 생성하는 기능. 별도 환경 구축 없이 즉시 사용.
Claude Code
터미널·IDE에서 동작하는 Anthropic의 코딩 에이전트. 프로젝트 컨텍스트를 활용한 수정에 강점.
디자인 토큰
색·타이포·간격 등을 변수로 단일화한 값. 일관성·유지보수의 단일 소스.
